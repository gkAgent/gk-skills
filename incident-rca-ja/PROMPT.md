## Phase 0: スコープの確認

まず以下を質問してください:

> 障害の概要を教えてください。
>
> **障害種別**: システム停止 / 機能不全 / データ不整合 / 性能劣化 / セキュリティインシデント
> **発生日時**: YYYY-MM-DD HH:MM（JST）
> **影響範囲**: 対象システム・影響ユーザー数・業務影響
> **現状**: 復旧済み / 対応中 / 調査中
> **暫定対応**: 実施済みの対応内容

## 障害種別ごとの分析観点

| 障害種別 | 追加チェック項目 |
|---|---|
| バッチ停止 | ジョブスケジューラの設定 / 前処理依存 / タイムアウト設定 |
| データ不整合 | トランザクション境界 / 並行処理 / 外部キー制約 |
| 性能劣化 | スロークエリ / インデックス / 接続プール枯渇 / メモリリーク |
| 認証エラー | DB 権限 / SSL 証明書期限 / パスワード期限 |
| セキュリティ | アクセスログ / 不審IPの特定 / 影響範囲の確定（情報漏洩有無） |

## 絶対禁止事項

- **根本原因を「ヒューマンエラー」で終わらせる禁止** — 「なぜヒューマンエラーが起きたか」まで掘り下げる（Why 4〜5まで）
- **暫定対応と恒久対応の混同禁止** — 区別して記載。暫定対応のみで終わらせない
- **再発防止策なし禁止** — 必ず具体的なアクション・期限・担当者を記載
- **途中停止禁止** / **AI voice禁止**
